# Telegram Proxy Checker

Программа для проверки и управления MTProto/HTTP/SOCKS5 прокси серверами Telegram.

## 🚀 Возможности

### Управление прокси
- ✅ **Добавление прокси** вручную (MTProto, HTTP, SOCKS5)
- ✅ **Импорт из HTML** файлов (экспорт из Telegram)
- ✅ **Добавление из ссылок** формата `tg://proxy?...`
- ✅ **Удаление выбранных** прокси
- ✅ **Автоматическое удаление дубликатов** (по host:port:secret)

### Проверка прокси
- ✅ **Проверка всех** прокси в списке
- ✅ **Проверка выбранных** (правой кнопкой мыши)
- ✅ **Авто-проверка** каждые 30 секунд
- ✅ **MTProto handshake** с парсингом секрета
- ✅ **Измерение пинга** (latency)
- ✅ **Адаптивная частота** проверки (0-1000)
- ✅ **Параллельная проверка** (до 100 потоков)

### Фильтрация и поиск
- ✅ **Поиск по имени сервера** (host)
- ✅ **Скрытие онлайн/оффлайн/неизвестных**
- ✅ **Сортировка** по клику на заголовок столбца
- ✅ **Редактирование частоты** прямо в таблице

### Экспорт и сохранение
- ✅ **Копирование ссылки** одного прокси
- ✅ **Копирование всех ссылок**
- ✅ **Автосохранение** в JSON файл
- ✅ **Открытие ссылки** двойным кликом по хосту

## 📋 Требования

- **.NET 6.0** или выше
- **Windows** (WinForms)

## 🔧 Установка

1. Склонируйте репозиторий или скачайте исходный код
2. Откройте проект в **Visual Studio 2022** или **VS Code**
3. Убедитесь, что установлена **.NET 6.0 SDK**
4. Соберите проект:
   ```bash
   dotnet build
   ```
5. Запустите:
   ```bash
   dotnet run
   ```

## 📖 Использование

### Добавление прокси

**Способ 1: Вручную**
1. Выберите тип прокси (MTProto/HTTP/SOCKS5)
2. Введите Host/IP
3. Укажите Port
4. Введите Secret (опционально)
5. Нажмите **Add**

**Способ 2: Из ссылки**
1. Вставьте ссылку вида `tg://proxy?server=...&port=...&secret=...`
2. Нажмите **Add Link**

**Способ 3: Импорт из HTML**
1. Нажмите **Import HTML**
2. Выберите файлы экспорта Telegram
3. Прокси автоматически добавятся с датами

### Проверка прокси

**Проверка всех:**
- Нажмите **Check All** для проверки всего списка
- Прогресс-бар покажет статус проверки
- Статус обновится: Online/Offline с пингом

**Проверка выбранных:**
1. Выделите нужные строки (Ctrl+клик для множественного выбора)
2. Нажмите **правую кнопку мыши**
3. Выберите **🔍 Check Selected**

**Авто-проверка:**
- Установите галочку **Auto Check (30s)**
- Программа будет проверять прокси каждые 30 секунд

### Фильтрация и поиск

**Поиск:**
- Введите часть имени сервера в поле **Search Host...**
- Таблица отфильтруется мгновенно

**Фильтры статуса:**
- **Hide Online** - скрыть рабочие прокси
- **Hide Offline** - скрыть нерабочие
- **Hide Unknown** - скрыть непроверенные

**Сортировка:**
- Кликните по заголовку любого столбца для сортировки
- Повторный клик меняет направление (↑/↓)

### Частота проверки (Frequency)

Каждому прокси назначается частота проверки (0-1000):
- **0** - никогда не проверять
- **1000** - проверять каждый цикл
- **500** - проверять с вероятностью 50%

**Адаптация:**
- При успешной проверке частота увеличивается
- При неудаче - уменьшается
- Можно редактировать вручную в колонке **Frequency**

### Контекстное меню (ПКМ)

- **🔍 Check Selected** - проверить выделенные
- **🔗 Copy Link** - скопировать ссылку выделенного
- **📋 Copy All Links** - скопировать все ссылки

## 📊 Структура данных

Программа сохраняет данные в `proxies.json`:

```json
{
  "Type": "MTProto",
  "Host": "example.com",
  "Port": 443,
  "Secret": "secret_key",
  "Status": "Online",
  "Latency": 150,
  "LastCheck": "2024-01-01T12:00:00",
  "DateAdded": "2024-01-01T10:00:00",
  "Frequency": 1000
}
```

## 🔍 Как работает проверка

1. **Подключение** к серверу через TCP (IPv4)
2. **Отправка handshake**:
   - Для порта 443: префикс `0xdddddddd`
   - Для остальных: префикс `0xeeeeeeee`
   - + 32 байта секрета (из hex)
3. **Ожидание ответа** (таймаут 3 секунды)
4. **Анализ**: если сервер ответил - он онлайн

## ⚙️ Настройки

- **MaxDegreeOfParallelism**: 100 (количество одновременных проверок)
- **AutoCheck Interval**: 30000 мс (30 секунд)
- **Timeout**: 3000 мс (3 секунды на проверку)

## 🐛 Устранение неполадок

**Прокси не проверяются:**
- Проверьте подключение к интернету
- Убедитесь, что файрвол не блокирует приложение
- Попробуйте уменьшить количество одновременных проверок

**Ошибка "No Data":**
- Сервер может требовать полный MTProto handshake
- Проверьте правильность секрета

**Программа зависает:**
- Нажмите **Stop** для остановки проверки
- Уменьшите количество прокси для одновременной проверки
